# 旅游攻略网站 - 项目设计文档

## 项目概述

本项目是一个基于 Next.js 框架开发的旅游攻略网站，旨在为用户提供旅游目的地信息、攻略分享和社区交流平台。项目使用最新的 Next.js 15 版本，采用 React 19 和 TypeScript 进行开发，并集成了 Tailwind CSS 进行样式设计。

## 技术栈

### 核心框架
- **Next.js 15.2.2**: 基于 React 的全栈框架，支持服务端渲染(SSR)、静态站点生成(SSG)和客户端渲染(CSR)
- **React 19.0.0**: 用户界面库
- **TypeScript 5+**: 提供类型安全的 JavaScript 超集

### 样式解决方案
- **Tailwind CSS 4**: 实用优先的 CSS 框架
- **PostCSS**: CSS 转换工具

### 开发工具
- **ESLint 9**: 代码质量检查工具
- **ESLint Config Next**: Next.js 特定的 ESLint 配置

## 技术要点

### 1. Next.js App Router
- 项目采用 Next.js 的 App Router 架构，使用基于文件系统的路由
- `src/app` 目录下的文件结构决定了应用的路由结构
- `page.tsx` 定义了路由页面
- `layout.tsx` 提供了共享布局和设计

### 2. TypeScript 集成
- 项目完全使用 TypeScript 开发，提高代码质量和开发体验
- 使用路径别名（@/*）简化导入语句
- 配置了严格的类型检查

### 3. 响应式设计
- 通过 Tailwind CSS 实现响应式设计
- 支持亮色/暗色模式切换，通过 CSS 变量实现

### 4. 部署和构建
- 支持通过 `next build` 构建生产版本
- 可轻松部署到 Vercel 平台

## 页面设计

### 目的地页面

我们针对目的地页面设计了一个创新的目录式布局，优化用户浏览体验：

#### 设计理念
目的地页面采用"左侧目录导航 + 右侧内容展示"的布局模式，类似于电子书或文档阅读器的界面。这种设计允许用户在不同的城市信息之间快速切换，无需频繁返回主页面，提高了信息获取效率。

#### 主要功能和特性

1. **按主题分类的城市目录**：
   - 不同于传统的地理区域分类，我们根据旅游主题对城市进行分组
   - 主题类别包括：经典必游、自然风光、历史名胜、绚丽都市、美食之都等
   - 同一个城市可以出现在多个主题分类中，方便用户根据兴趣浏览

2. **搜索功能**：
   - 页面顶部配备搜索框，支持快速查找城市
   - 搜索结果实时显示，提高用户体验

3. **动态内容加载**：
   - 用户点击左侧城市名称时，右侧内容区域会动态更新所选城市的详情
   - 使用 Next.js 的客户端路由功能，通过 URL 查询参数（如 `/destinations?city=beijing`）保留用户选择状态
   - 切换城市时不会刷新整个页面，仅更新内容区域，提供无缝体验

4. **响应式布局**：
   - 在桌面设备上展示完整的双栏布局
   - 在移动设备上自动调整为垂直布局，保持良好的用户体验

5. **主题展开/折叠**：
   - 可折叠的主题分类，帮助用户在众多选项中保持界面整洁
   - 展开/折叠状态支持本地存储，记住用户的浏览偏好

#### 技术实现

1. **React 状态管理**：
   - 使用 React 的状态钩子（useState, useEffect）管理搜索、选择和展开/折叠状态
   - 通过 URL 查询参数共享和保持状态

2. **主题分类数据结构**：
   - 使用 TypeScript 接口定义主题和城市之间的关系
   - 主题数据集中存储并支持多语言

3. **内容模板**：
   - 使用通用的 `DestinationTemplate` 组件渲染城市内容
   - 支持多语言内容和动态加载

4. **搜索和过滤**：
   - 基于城市名称的实时搜索过滤
   - 支持多语言搜索

#### 用户体验优化

1. **视觉反馈**：
   - 当前选中的城市高亮显示
   - 展开/折叠图标清晰指示当前状态

2. **无缝导航**：
   - 用户可以在不同城市间快速切换，无需页面刷新
   - 保持滚动位置，避免浏览中断

3. **多语言支持**：
   - 城市名称、主题分类和页面文本均支持多语言显示
   - 与网站的全局语言设置集成

4. **渐进式增强**：
   - 基本功能不依赖 JavaScript，确保可访问性
   - 添加交互增强功能，提升使用体验

这种设计特别适合包含大量城市信息的旅游网站，帮助用户高效地发现和探索不同的旅游目的地。
